# Component Refs

All components that it makes sense to do so (Mostly inputs, dropdowns, etc...) expose a `ref` prop to get access to their underlying DOM node. Note that for things like the `TextInput` component, you will get access to the input DOM node, not any of the wrapper elements.

The current list of components that expose a `ref` prop are (clickable links):

- [`TextInput`](?path=/docs/inputs-textinput--overview)
- [`Textarea`](?path=/docs/inputs-textarea--overview)
- [`Select`](?path=/docs/dropdowns-select--overview)
- [`NumberInput`](?path=/docs/inputs-numberinput--overview)
- [`Checkbox`](?path=/docs/inputs-checkbox--overview)
- [`SearchInput`](?path=/docs/inputs-searchinput--overview)
- [`ToggleSwitch`](?path=/docs/inputs-toggleswitch--overview)
